# Blue Archive GUI Automation Test

## 프로젝트 개요
- **목적**: Blue Archive Normal 1-4 스테이지를 대상으로 한 GUI 기반 자동화 테스트
- **방식**: Live 빌드 환경에서 화면 인식만 사용 (메모리/패킷 조작 없음)
- **기술 스택**: Python 3.11, pyautogui, OpenCV Template Matching, Tesseract OCR(부분 사용)

---

## 테스트 범위 요약
본 프로젝트는 **"화면에 보이는 결과가 정상적으로 노출되었는가"** 만을 검증 대상으로 하며,
수치 정확성, 서버 데이터, 내부 상태 값은 테스트 범위에서 제외합니다.

---

## 테스트 케이스 목록

### TC-001: 발판 이동 검증
- **목적**: 캐릭터가 실제로 발판 이동을 수행했는지 확인
- **접근법**:
  - 이동 가능한 발판 템플릿 매칭
  - 이동 전/후 캐릭터 마커 위치 비교
- **PASS 기준**:
  - 캐릭터 마커 위치가 10px 이상 변경됨
- **FAIL 기준**:
  - 이동 후 위치 변화 없음
  - 이동 불가능 발판 클릭

---

### TC-002: 전투 진입 검증
- **목적**: 적 발판 클릭 후 전투 화면으로 정상 전환되었는지 확인
- **접근법**:
  - 전투 UI 관련 템플릿 3종 중 2개 이상 매칭
- **PASS 기준**:
  - 전투 UI 조건 2개 이상 인식
- **FAIL 기준**:
  - 제한 시간 내 전투 UI 미검출

---

### TC-003: 학생별 EX 스킬 사용 시도
- **목적**: 각 학생의 EX 스킬 사용 인터랙션이 수행되었는지 확인
- **접근법**:
  - 고정 좌표 기반 스킬 버튼 드래그 자동화
  - 슬롯별 사용 시점 로그 기록
- **PASS 기준**:
  - 각 슬롯에 대해 드래그 동작 수행 및 로그 기록
- **FAIL 기준**:
  - 드래그 동작 실패 또는 로그 미기록

---

### TC-004: 코스트 소모 검증
- **상태**: 테스트 범위 제외
- **제외 사유**:
  - OCR 정확도 한계로 신뢰성 확보 불가

---

### TC-005: 전투 결과 데미지 리포트 검증
- **목적**: 전투 참가 학생 전원이 데미지 리포트 UI에 매핑되었는지 확인
- **접근법**:
  - 데미지 리포트 화면에서 학생 아이콘 템플릿 매칭
- **PASS 기준**:
  - 전투 참가 학생 아이콘이 데미지 리포트 UI에 존재
- **FAIL 기준**:
  - 참가 학생 중 1명 이상 UI 미존재

---

### TC-006: 보상 획득 UI 검증
- **목적**: 전투 종료 후 보상 UI가 정상 노출되었는지 확인
- **접근법**:
  - Victory → Mission Complete → 보상 아이콘 순차 검증
- **PASS 기준**:
  - 보상 아이콘 1개 이상 화면에 존재
- **FAIL 기준**:
  - 보상 UI 미노출 또는 아이콘 검출 실패

---

## 비검증 항목 명시
- 데미지 수치 / 순위
- 코스트 수치 및 변화량
- 아이템 수량 증가
- 서버 데이터 반영 여부

---

## 한계 및 전제
- 해상도: **2560x1440 고정**
- 모든 검증은 **화면 UI 존재 여부 기준 (Yes / No)**
- 게임 UI 특성상 OCR 기반 검증은 최소화

---

## 결론
본 자동화는 **Live 환경에서 안정적으로 검증 가능한 최소 단위(UI 존재)**에 초점을 맞춘 테스트이며,
과도한 수치 검증 대신 실패 원인이 명확한 구조를 지향합니다.
